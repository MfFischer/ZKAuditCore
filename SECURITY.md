# Security Policy

## Supported Versions

This repository currently supports the active `main` branch for security updates.

## Reporting a Vulnerability

Please do not disclose vulnerabilities publicly before coordinated remediation.

Report security concerns by opening a private security advisory on GitHub or contacting the maintainers directly with:

- Impact summary
- Reproduction steps
- Affected files or interfaces
- Suggested mitigations (if available)

## Response Targets

- Initial triage acknowledgement: within 3 business days
- Severity assessment and remediation plan: within 7 business days
- Patch target for high/critical issues: as soon as safely possible

## Scope

Security reports are especially valuable for:

- Incorrect or misleading solver/rule evidence attribution
- Non-deterministic output that can break reproducibility guarantees
- Coverage overstatement or exclusion misclassification
- Signature and attestation verification weaknesses
