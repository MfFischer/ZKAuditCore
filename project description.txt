Project Vision & Positioning
Name: ZK-AuditCore
Vision: A deterministic, coverage-aware, exploit-driven ZK auditing pipeline that produces verifiable, reproducible audit artifacts. Augments cryptographers with proof-backed analysis, not probabilistic guesses.
Positioning: Trust infrastructure for ZK audits. Not an AI replacement, but a force multiplier that guarantees what was analyzed, how it breaks, and why findings are defensible.
🏛️ Core Architecture (5 Pillars)
Pillar
Responsibility
Key Constraint
1. Deterministic Analysis Engine
Static rules + SMT verification + constraint graph traversal
LLM never performs math or constraint reasoning
2. Coverage & Attestation Layer
Constraint/property coverage metrics, version-pinned execution, Sigstore signing
Coverage excludes trivial/pass-through constraints; methodology is public
3. Exploit Discovery Pipeline
Counterexample expansion, proof-failure detection, protocol mismatch analysis
Exploits require verifier loop or formal counterexample, not pattern matching
4. LLM Orchestrator
Strict JSON/tool-routing, proof-grounded reporting, constraint tracing
Zero cryptographic generation; temperature=0.1; schema-validated outputs only
5. Auditor Workflow & Integration
CLI-first, GitHub CI/CD, diff views, annotation system, plugin ecosystem
Designed for human-in-the-loop sign-off, not autonomous execution

Marketplace for rule/property plugins + audit firm partnerships
🛡️ Execution Guardrails (Non-Negotiables)
LLM stays in orchestration mode only → Never generates constraints, proofs, or mathematical derivations. All outputs trace to solver/rule IDs.
SMT partitioning is dependency-aware → Naive splitting breaks invariants. Use constraint graph to isolate; fallback to UNKNOWN with exact unsatisfied subset.
Coverage is defensible, not vanity → Exclude pass-through signals. Publish methodology. Map % verified to actual constraint subsets.
Fuzzing starts deterministic → Constraint-violation + edge cases first. Proof-failure fuzzing requires a live verifier loop.
IR is an analysis API, not a monolith → Each framework compiles to a common query/coverage schema. Avoid lossy cross-framework translation early.
Human sign-off is mandatory → System outputs Verified / Tested / Untested. Auditor approves, annotates, or escalates.
